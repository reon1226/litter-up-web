# Information Architecture

## Purpose

This document defines how users move through the Litter Up application.

The goal is to create the simplest possible experience while keeping every step meaningful.

Every screen must have one clear purpose.

---

# Main Flow

Home

↓

Upload Artwork

↓

Generate Pixel Art

↓

Generate Numbered Blueprint

↓

Review Color Palette

↓

Upload Litter Photos

↓

Automatic Color Classification

↓

Review Classification

↓

Generate Photo Mosaic

↓

Download Results

---

# Screen Structure

## 1. Home

Purpose:
Introduce the project and allow users to start immediately.

Main Components:

- Hero Section
- "Start Creating" Button
- Short explanation
- Example artwork
- Footer

---

## 2. Artwork Upload

Purpose:
Upload the original artwork.

Functions:

- Drag & Drop
- File Picker
- Image Preview

Supported formats:

- JPG
- JPEG
- PNG
- WEBP

---

## 3. Pixel Art Preview

Purpose:

Display the generated 20-color pixel art.

Functions:

- Zoom
- Compare with original
- Regenerate

---

## 4. Blueprint

Purpose:

Display numbered blueprint.

Functions:

- Number overlay
- Toggle numbers
- Zoom
- Download Blueprint

---

## 5. Color Palette

Purpose:

Explain every generated color.

Each color displays:

- ID
- Preview
- RGB
- HEX
- Percentage

---

## 6. Photo Upload

Purpose:

Collect litter photographs.

Functions:

- Drag & Drop
- Folder Upload
- Multiple Upload
- Upload Progress

---

## 7. Classification

Purpose:

Show automatic color assignment.

Functions:

- Assigned Color
- Representative Color
- Manual Change
- Filter by Color

---

## 8. Mosaic Preview

Purpose:

Preview the completed artwork.

Functions:

- Zoom
- Fullscreen
- Before / After
- Randomize

---

## 9. Download

Purpose:

Export generated files.

Available Downloads:

- Photo Mosaic
- Blueprint
- Palette
- Classification List

---

# Navigation Rules

Users should always know:

- Where they are
- What happens next
- How to go back

No screen should require more than one primary action.

---

# UX Principles

The application should feel:

- Fast
- Simple
- Creative
- Inspiring

Users should never feel confused.

---

# Design Philosophy

Every screen should answer one question:

"What is the next obvious action?"

If users hesitate,

the interface should be simplified.
